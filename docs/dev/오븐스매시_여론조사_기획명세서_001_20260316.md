# 프로젝트 명세서: 오븐스매시 갤러리 여론 분석 시스템 (DC-Sentiment)

## 1. 개요
- **목표:** 디시인사이드 '쿠키런 오븐스매시' 마이너 갤러리의 게시글을 분석하여 일별/주간 유저 여론 변화를 파악.
- **주요 전략:**
    - **하이브리드 분석:** LLM(OpenAI)을 통한 매일의 신조어/문맥 학습 + 로컬 라이브러리를 통한 고속 전수 조사.
    - **에이전트 구조:** PM, Backend, Frontend, QA 역할 분담 및 기능별 모듈화.

## 2. 프로젝트 조직 및 역할 (Roles)
| 역할 | 담당 주체 | 주요 임무 |
| :--- | :--- | :--- |
| **PM** | Antigravity | 전체 기획, 명세서 관리, 마일스톤 관리 |
| **Backend** | Antigravity (BE) | 크롤러 개발, DB 설계, 분석 엔진 구축, API 연동 |
| **Frontend** | Antigravity (FE) | 분석 결과 시각화 (Markdown/HTML 리포트) |
| **QA** | Antigravity (QA) | 테스트 케이스 작성, 데이터 정합성 검증, 에러 핸들링 점검 |

## 3. 기능 요구사항 (Requirements)
### R1. 데이터 수집 (Crawler)
- 대상: `https://gall.dcinside.com/mgallery/board/lists/?id=ovensmash`
- 수집 항목: 게시글 번호, 제목, 작성일, 조회수, 추천수.
- 규칙: 
    - 첫 실행 시 최근 1주일치 데이터 소급 수집.
    - 이후 매일 정해진 시간에 신규 데이터 추가 수집.
    - 차단 방지를 위한 User-Agent 설정 및 랜덤 딜레이 적용.

### R2. 여론 분석 (Sentiment Analysis)
- **Step A (Daily Tutor):** 매일 샘플 게시글(50개)을 OpenAI API에 전달하여 오늘의 신조어 및 감성 스코어 맵(Map) 생성.
- **Step B (Local Engine):** 생성된 맵을 바탕으로 로컬에서 전체 게시글 감성 분석 (긍정/부정/중립).
- **결과값:** `-1.0(매우 부정)` ~ `1.0(매우 긍정)` 사이의 스코어 산출.

### R3. 데이터 저장 (Storage)
- 기술: SQLite (경량화 및 파일 기반 관리 용이).
- 테이블 구성: `posts` (수집 데이터), `lexicon` (감성 사전), `summary` (일별 통계).

### R4. 리포트 (Reporting)
- 일별 요약 리포트 (Markdown 형식).
- 주요 키워드 트렌드 Top 5 추출.

## 4. 기술 스택 (Tech Stack)
- **Language:** Python 3.10+
- **Library:** 
    - Crawler: `requests`, `BeautifulSoup4`
    - AI: `openai` (GPT-4o-mini)
    - DB: `sqlite3`
    - Analysis: `pandas`
- **Infrastructure:** Local OS (Windows)
