## 2. 문제 정의

### 증상
긍정 대표글 Top10에서 "이겜은 진째 기대가 안되네"가 1위로 노출됨.

### 근본 원인
`sentiment_analyzer.py`의 `analyze_locally()`가 단순 키워드 매칭만 수행:
- "기대" → 사전 점수 +0.5 적용
- 뒤따라오는 "안되네"는 감성 점수에 영향 없음
- 결과: 부정 문장이 +0.5(긍정)으로 분류됨

### 오분류 패턴 분류
| 패턴 | 예시 | 기존 점수 | 올바른 분류 |
|------|------|-----------|-------------|
| 긍정어 + 뒤 부정어 | "기대가 안되네" | +0.5 | 부정 |
| 앞 부정어 + 긍정어 | "안 재밌다" | +0.7 | 부정 |
| 없다 복합 (사전 누락) | "재미없다" | 0.0 | 부정 |

## 3. 수정 내용

### 수정 파일
`sentiment_analyzer.py`

### A. `analyze_locally()` — 부정어 양방향 window 처리

- 클래스 변수 `NEGATION_WORDS` 추가
- 키워드 매칭 시 **앞 6글자 + 뒤 6글자** window 내에 부정어가 있으면 점수 반전(`-s`)
- 기존 `matched_positions` 긴 키워드 우선 매칭 로직 유지

```
"기대가 안되네"
    ↑ idx=0, word_end=2
    following = "가 안되네" → "안되" 감지 → -0.5 (반전)
```

### B. `_init_default_lexicon()` — 복합 부정 표현 추가

붙여쓰기 패턴 및 window로 잡기 어려운 형태 직접 등록:
- "재미없다" → -0.7
- "재미없어", "재미없네", "재미없음", "재미없는" → -0.6
- "기대안됨", "기대가안됨", "기대없다", "기대없어" → -0.5 ~ -0.6
- "안재밌다", "안재밌어", "안재밌네" → -0.6 ~ -0.7
- "못즐기", "못하겠다" → -0.5

### C. `update_lexicon_with_llm()` — LLM 프롬프트 보강

가이드라인에 부정어 패턴 분류 지침 추가:
- "안 + 긍정어", "못 + 긍정어" 조합 → 부정(-0.5 ~ -0.7)
- "없다" 붙는 부정 복합어 → 부정(-0.5 ~ -0.7)
