# 프로젝트 명세서 (다중 갤러리 스케줄러 아키텍처 개편)
**버전:** v1.0
**날짜:** 2026-03-19

## 1. 개요
* 디시인사이드 갤러리 여론조사 프로그램(`DC_search`)의 사용자 동기적 요청 크롤링 방식을 전면 개편한다.
* 다수의 사용자가 동시에 검색 시 발생하는 서버 부하 및 IP 밴 현상을 원천적으로 차단하기 위해, **백그라운드 스케줄링** 방식으로 전환한다.

## 2. 대상 갤러리 (초기 타겟 3종)
1. 이름: 쿠키런 / ID: `cookierun` / 타입: 마이너
2. 이름: 쿠키런 킹덤 / ID: `cookierunkingdom` / 타입: 메이저
3. 이름: 쿠키런 오븐 스매시 / ID: `ovensmash` / 타입: 마이너

## 3. 구조 변경 계획 (설계 상세)
### 3-1. 설정 파일 도입 (`config.py`)
* 연동할 타겟 갤러리 정보가 하드 코딩된 딕셔너리 리스트 생성. 향후 타 사이트/목록 확장 시 활용됨.

### 3-2. 백그라운드 수집 데몬 (`scheduler.py`)
* 웹 서버(`server.py`)와 독립적으로 터미널에서 구동되는 메인 데이터 수집 루프.
* 주기(예: 30분)마다 3개의 갤러리를 순회 방문하여 `run_daily_process()` 호출.
* 디시인사이드 IP 블랙 차단을 막기 위해, 한 갤러리 수집 후 다음 갤러리 수집 전 약 3~5초 등의 지연 시간(Sleep) 삽입.

### 3-3. 웹 서버 간소화 (`server.py`)
* 사용자의 자의적 분석 요청 기능(`/analyze`의 POST 요청 등) 라우터 삭제.
* 메인 페이지(`/`) 접속 시, 설정된 기준 갤러리 항목 중 넘어온 `gallery` 인자값을 필터링하고 DB만 단순 SELECT 하여 빠르게 렌더링.
* 오직 읽기 전용으로 백엔드가 전환됨.

### 3-4. 프론트엔드 개편 (`templates/dashboard.html`)
* 중앙의 직관적 URL 검색 폼 삭제.
* 갤러리 3종 중 조회할 대상을 고를 수 있는 Select 컴포넌트로 UI를 교체.
