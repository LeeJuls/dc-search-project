# 프로젝트 히스토리: DB 저장 및 통합 관리 (Phase 3)

## 📅 일시: 2026-03-16
## 📜 버전: v1.2

### 1. 개발 내용
- **모듈명:** `db_manager.py`, `main.py`
- **구현 기능:**
    - `DBManager`: SQLite 테이블(posts, lexicon, daily_summary) 생성 및 관리.
    - **중복 방지:** `INSERT OR IGNORE`와 `post_num` Unique 제약 조건을 통해 이미 저장된 글은 건너뜀.
    - **기간 필터링:** `main.py`에서 수집 기준일(기본 7일)을 설정하여 필요한 범위의 데이터만 수집.
    - **통합 프로세스:** 크롤러 -> 데이터 정제 -> DB 저장 -> 결과 보고의 워크플로우 완성.

### 2. 테스트 결과 (QA)
- **1차 실행:** 1주일치 기준 데이터를 정상 수집하여 DB(`dc_sentiment.db`)에 44건 저장 완료.
- **2차 실행 (중복 테스트):** 동일한 조건으로 실행 시 "새로운 게시글 0개 반영" 메시지 확인 -> 중복 방지 로직 정상 작동 확인.
- **데이터 무결성:** 수집된 게시글의 번호, 제목, 작성일 등이 DB에 정확히 매칭됨을 확인.

### 3. 향후 과제
- **Phase 4:** OpenAI API 연동을 통한 지능형 감성 사전 생성 로직 추가.
- **API 키 요청:** LLM 분석 단계 진입을 위해 사용자에게 API 키 요청 필요.
