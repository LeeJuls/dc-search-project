# 프로젝트 히스토리: 기획 및 팀 구성 확정

## 📅 일시: 2026-03-16
## 📜 버전: v1.0

### 1. 의사결정 사항
- **조직 구성:** PM, Backend, Frontend, QA의 4인 체제 페르소나 도입.
- **분석 전략:** 전수 조사의 비용 부담을 줄이고 정확도를 높이기 위해 'Daily LLM Tutor + Local Sentiment Engine' 하이브리드 방식 채택.
- **데이터 저장:** 확장성보다 관리 편의성을 우선하여 SQLite 사용 결정.

### 2. 논의 내용
- 사용자가 에이전트 구성을 넘어서 PM/개발자/QA라는 실무 조직 구조를 제안함.
- API 단독 사용보다 신조어 업데이트를 통한 로컬 라이브러리 혼합 방식이 효율적이라는 점에 합의함.

### 3. 향후 과제
- OpenAI API 키 확보 및 환경변수 설정 가이드 제공.
- 디시인사이드 크롤러의 안정성(차단 방지) 확보를 위한 QA 테스트 진행.
