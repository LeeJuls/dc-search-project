# 프로젝트 히스토리 (전체 코드 리뷰 및 버그 수정)
**버전:** v1.0
**날짜:** 2026-03-19

## 1. 배경 및 발단

라이브 서버(Render)에서 대시보드 접속 시 "데이터를 수집 및 집계 중입니다." 메시지만 표시되는 문제가 지속되었다.
오너의 전체 코드 리뷰 요청으로 원인 진단 및 다수의 버그 수정 작업을 진행하였다.

---

## 2. 주요 의사 결정 사항

### (1) 감성 사전 방향 전환
욕설 포함 여부 → **게임 플레이 경험 기반** 키워드로 전환하기로 합의.
- "욕설이 들어가도 좋은 반응일 수 있고, 욕설 없어도 부정일 수 있다"는 커뮤니티 맥락 반영
- "사기캐", "밸붕", "갓겜", "혜자", "무과금" 등 게임 유저 고유 어휘 중심으로 재구성
- LLM 프롬프트에 욕설 판단 금지 가이드라인 명시

### (2) 키워드 매칭 우선순위 정책
"무과금 ㅈ망겜" 같은 표현에서 "과금"이 먼저 매칭되어 긍정/부정 혼동 발생 사례를 오너가 제기.
→ **긴 키워드 우선 매칭** 정책 채택 (예: "무과금" > "과금" 순으로 탐색)

### (3) Supabase 1000행 한계
오너 질문으로 발견. Supabase 기본 페이지네이션이 1000행 제한.
→ 모든 쿼리에 `.limit(5000)` 명시 적용으로 단기 대응.
→ 장기적으로 데이터가 5000행 초과 시 페이지네이션 구현 필요 (향후 과제)

### (4) 날짜 필터 방식
"1일" 조회 시 오전 시간대에는 전날 오전 게시글이 누락됨을 발견.
→ 롤링 24시간 방식 → **자정(00:00) 기준** 방식으로 변경 합의

## 4. 발견된 인프라 이슈

### Supabase 이중 프로젝트 혼동
- Supabase에 구글 계정(Google login)과 깃허브 계정(GitHub login) 각각으로 프로젝트가 생성되어 있었음
- 오너가 대시보드에서 확인하던 프로젝트와 `.env`의 프로젝트가 다를 수 있다는 의심이 제기됨
- 확인 결과: `.env`의 `SUPABASE_URL=https://iwyadjubhywkbzgtrlwr.supabase.co`가 실제 데이터가 있는 정확한 프로젝트를 가리키고 있음을 확인
- Supabase 에디터에서 데이터 정상 확인, Render 환경변수도 동일 프로젝트로 설정됨 확인

### GitHub Actions 크롤링 실패
- GitHub Actions ubuntu-latest 러너에서 DC Inside 크롤링 시 `ConnectTimeoutError` 지속 발생
- DC Inside가 클라우드 서버 IP 차단하는 것으로 추정 (방어 정책)
- OpenRouter 무료 모델 429 rate limit도 간헐적 발생
- 크롤링 실패해도 기존 DB 데이터로 대시보드는 정상 서비스됨

## 5. 미해결 및 향후 과제

| 항목 | 설명 | 우선순위 |
|------|------|---------|
| GitHub Actions 크롤링 실패 | DC Inside의 클라우드 IP 차단 대응 방안 필요 (프록시, 우회 등) | 높음 |
| 5000행 초과 시 페이지네이션 | `.limit(5000)` 임시 대응 중, 장기적으로 커서 기반 페이지네이션 필요 | 중간 |
| OpenRouter 429 대응 | 무료 모델 Rate Limit 초과 시 재시도 로직 또는 유료 전환 검토 | 낮음 |
