# 히스토리: 감성 분석 부정어 처리 개선

## 📅 일시: 2026-03-25
## 📜 버전: v1.0

---

### 1. 발단
긍정 대표글 Top10에서 "이겜은 진째 기대가 안되네"가 1위로 노출되는 오분류 버그 발견.
명백한 부정 표현이 긍정으로 분류된 것으로, 감성 분석 신뢰도에 심각한 영향을 줌.

### 2. 원인 분석

**`analyze_locally()` 단순 키워드 매칭의 한계:**
- "기대" 키워드 → 사전 점수 +0.5 적용
- "안되네"는 별도 처리 없음 → 부정 문맥 무시
- 기존 구조는 복합 부정어("안뜨", "못뽑")만 원자 단위로 처리, 동적 부정어 조합 불가

**오분류 패턴 3종 확인:**
1. 긍정어 + 뒤 부정어: "기대가 안되네"
2. 앞 부정어 + 긍정어: "안 재밌다"
3. 복합 부정 표현 사전 누락: "재미없다" → 0.0(중립)

### 3. 의사결정

**A안 (채택): 양방향 window 부정어 탐지 + 사전 보강**
- `analyze_locally()`에서 키워드 앞뒤 6글자 window 탐색
- `NEGATION_WORDS` 클래스 변수로 부정어 목록 관리
- 복합 부정 표현 사전 직접 등록으로 이중 방어

**B안 (미채택): 형태소 분석기(KoNLPy) 도입**
- 정확도는 높지만 의존성 증가, 실행 속도 저하
- 현재 프로젝트 규모에 과도한 복잡도

### 4. 구현 내용

#### `sentiment_analyzer.py` 수정
- `NEGATION_WORDS` 클래스 변수 추가 (15개 부정어 패턴)
- `analyze_locally()`: 앞 6글자 + 뒤 6글자 window에서 부정어 탐지 시 점수 반전
- `_init_default_lexicon()`: 복합 부정 표현 11개 추가
- `update_lexicon_with_llm()`: LLM 프롬프트 가이드라인 2줄 보강

### 5. 테스트 결과

**인라인 테스트 10/10 PASS**

| 제목 | 수정 전 | 수정 후 |
|------|---------|---------|
| "이겜은 진째 기대가 안되네" | +0.50 긍정 | **-0.50 부정** |
| "요즘 안 재밌다" | +0.70 긍정 | **-0.70 부정** |
| "재미없다 진짜" | 0.00 중립 | **-0.70 부정** |
| "기대된다 이번 패치" | +0.50 긍정 | +0.50 긍정 (유지) |
| "안뜨/못뽑 등 원자 부정어" | 부정 | 부정 (유지) |

### 6. 발견된 이슈 및 주의사항
- 부정어 window 크기(6글자): 한국어 조사 최대 2자 + 공백 + 부정어 3자 기준으로 설정
- "안 써도 된다" (긍정 문맥) 등 일부 복잡한 부정어 반전이 잘못 동작할 수 있음 → 향후 모니터링 필요
- 실서버 적용 전 `python reanalyze.py` 실행 필수 (DB 전체 재분석)

### 7. 향후 과제
- 부정어 오반전 케이스("안 써도 됨" 등) 발견 시 예외 처리 추가
- LLM이 학습한 신조어 중 복합 부정 패턴 사전 반영 모니터링
- 테스트 파일(`test_sentiment.py`) 공식 추가 검토
