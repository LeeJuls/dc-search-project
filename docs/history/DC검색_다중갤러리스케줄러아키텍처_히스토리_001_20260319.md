# 프로젝트 히스토리 (다중 갤러리 스케줄러 아키텍처 도입)
**버전:** v1.0
**날짜:** 2026-03-19

## 1. 개요 및 의사 결정 논의 사항
* **동기**: 사용자가 웹 폼을 통해 직접 검색을 실행하면 수백~수천 개의 게시글을 스크래핑/분석해야 하여 서버 부하가 심했습니다. 특히 다수 사용자가 동시 시도 시 DC 서버 IP 차단 및 DB 잠금 현상 발생 가능성이 제기되었습니다.
* **해결 및 의사 결정**: 실시간 수동 크롤링을 완전히 폐지했습니다. 사전에 정의된 타겟 갤러리(쿠키런 계열 3개)에 대해서만, 분리된 백그라운드 데몬(scheduler.py)이 고정 주기로 우회 탐색하도록 설계를 전환하기로 사용자와 합의하여 반영했습니다.

## 2. 세부 개발 및 코드 변경 기록
1. **[NEW] config.py**: 하드코딩된 환경 설정 파일 도입을 통해 지원하는 메이저/마이너 갤러리 통제 구축 (`cookierun`, `cookierunkingdom`, `ovensmash`).
2. **[NEW] scheduler.py**: 30분마다 대상 갤러리를 차례차례 방문. IP 차단 방지를 위해 작업 사이 10초 대기(Delay) 로직 추가.
3. **[MODIFY] server.py**: `POST /analyze` 엔드포인트 파기. 로딩 상태 없는 `GET /` 초고속 데이터 렌더링 구조로 변경하여 프론트/백엔드 분리 완성 효율 개선.
4. **[MODIFY] dashboard.html**: 자유형 텍스트 Input을 `<select>`(드롭다운 박스) 모듈로 교체 및 하드코딩된 목록 바인딩 처리 추가 완료.
