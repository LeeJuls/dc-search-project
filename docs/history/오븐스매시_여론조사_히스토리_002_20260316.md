# 프로젝트 히스토리: 프로그램 복구 및 AI 모델 최적화

## 📅 일시: 2026-03-16
## 📜 버전: v1.1

### 1. 의사결정 사항
- **AI 모델 변경:** 기존 `gemini-2.0-flash-exp:free` 모델의 불안정성(404 에러)으로 인해 안정적인 `llama-3.1-8b-instruct:free` 모델로 교체 결정.
- **코드 안정성 강화:** `main.py` 내의 모듈 임포트 구조 최적화 및 `os` 모듈 전역 임포트 적용.

### 2. 논의 내용
- 프로그램이 갑자기 종료된 원인을 파악한 결과, OpenRouter API의 모델 엔드포인트 누락과 환경 변수 처리 과정의 미흡함이 발견됨.
- 사용자의 요청에 따라 즉시 복구 작업을 진행하고, 가상환경(venv) 내의 안정적인 실행 환경을 확인 완료함.

### 3. 향후 과제
- Phase 6 (전체 자동화 및 최종 테스트) 진행.
- 크롤러의 차단 방지 로직이 실제 운영 환경에서 잘 작동하는지 장기 모니터링 필요.
